# 周氏十年断命学习库 V2.16.1

本地优先的个人案例学习工具。案例数据库、答题记录、个人笔记和你合法持有的参考书不会上传到服务器。

## 启动

双击项目根目录中的 `start_web.cmd`，浏览器会打开 `http://localhost:5173`，并自动读取：

```text
项目根目录\db\zhou.sqlite
```

若浏览器没有自动打开，可手动访问上述地址。也可以在项目根目录执行 `npm run dev`。

## 推荐学习顺序

1. 学习 → 基础表：十天干阴阳五行、十神速查、天干五合与生克、地支合会冲刑害破、十二地支藏干、六十甲子纳音；完成12道关系选择题。
2. 学习 → 案例选择题 → 入门 14 例：先形成稳定的认盘习惯。
3. 学习 → 专题步骤：按婚恋、事业、财富、学业等问题查看参数与判断顺序，每一步回到相关命例核对。
4. 点击“抽取专题盲练”：系统优先抽取尚未练过的同主题本命八字，只显示命盘；按五行模板写完再揭示原文。
5. 揭示后评估掌握度，按 1、3、7、14 或 30 天安排复习；也可继续抽取同专题下一例。
6. 切换“全部 130 例”：逐步覆盖所有具有完整命盘的案例。
7. 熟悉后再进入深度训练、规则证据和预留盲测。

每例 4–5 题，核对日主、日支藏干、日柱纳音和资料库方法标签；命盘存在无歧义的表内干支关系时增加 1 题，没有方法标签的 11 例改问命盘月柱。原训练集中的例117缺少完整命盘，只保留在深度训练。题目不会生成新的命理结论。

题库完成度始终按当前版本的实际题目重新计算；旧版记录若尚未补做新增题，会显示“待补题”而不是错误标记为完成。完成题组后，学习页按日主、地支藏干、纳音、干支关系、案例方法和月柱显示个人错项；基础项目可直接跳回对应速查表。

“待复习”列表直接进入错题重做，只显示上次答错的题，已答对项目保持不动。错题改正后会重新计算并保存整组成绩；全部改正的案例自动退出待复习列表。完成页仍保留“全部重做”。

“今日学习”的顺序为：已到期的案例间隔复习 → 尚未改正的选择题错项 → 下一组未完成的新题。到期案例先进入只看命盘的独立复盘，写下判断后才揭示原文；这样既不会把已有错误拖到全部 130 例之后，也不会用直接重读原文代替回忆练习。

普通案例的命盘下可以展开“基础拆解”和“干支关系提示”。基础拆解只列日主阴阳五行、月支与日支藏干、日柱纳音和年/月/时干相对日主的十神；不自动判断旺衰、喜忌或事件。选择题答完后才显示这些提示，深度训练、专题盲练与规则预留盲测在揭示前完全关闭提示。

## 日常入口

- 今日：自动续接未完成题组，全部完成后优先复习未全对案例。
- 学习：入门题组、全部案例、待复习、基础表、八个专题步骤、专题盲练和深度训练。
- 案例：智能检索并阅读原文。
- 书架：23 项核验书目与本机文件入口，其中陆致极体系 15 项（含独著、合著与版本参考）。

书架只整理书目、版本关系与学习顺序；需购入实体书后阅读，本站不提供电子书正文。

## 参考书架

书架分为陆致极体系、古籍与校注、近现代参考。陆致极体系保留 15 项已核验书目，并明确标出早期本、增订本、新版合集和合著关系；这个数字是书目条目数，不等同于内容互不重复的独立作品数。书架可切换“学习顺序”和“出版脉络”。将合法持有的 PDF、EPUB、MOBI 或 AZW3 放入 `library` 或任意子目录，例如：

```text
项目根目录\library\陆致极
项目根目录\library\古籍与校注
项目根目录\library\近现代参考
```

重新打开网页后，书架会按书名匹配并显示“本机阅读”。本地电子书由 `.gitignore` 排除，只允许 `localhost` 访问。

## 手机使用

电脑与手机连接同一 Wi-Fi，并保持启动窗口开启。在手机浏览器打开终端显示的 `http://192.168.x.x:5173`，再从手机文件中选择一份 `zhou.sqlite`。电脑不会通过局域网发送案例数据库或参考书。

## 数据与备份

- 案例数据库副本、答题记录和笔记保存在当前浏览器 IndexedDB。
- 设置中可导出或导入个人记录 JSON；同 ID 记录覆盖，其余记录保留，旧版备份仍可导入。
- 替换数据库不会覆盖个人记录。
- 清理浏览器站点数据前，请先导出记录。

## 开发与校验

```powershell
cd "项目根目录"
npm install --prefix web
npm run typecheck
npm run build
cd web
npm run verify
cd ..
py build\validate.py
```

`npm run verify` 会构建正式页面，核对藏干、纳音、十神与干支关系等基础表，再启动临时本地预览，自动检查数据库载入、旧题进度兼容、盲练提示边界、错题重做、备份边界、主要路由和 390 像素手机布局。构建产物位于 `web/dist/`，不包含本地案例原始资料和参考书文件。

## 发布 GitHub Pages

仓库只发布网页程序，数据库和本地资料不会进入 Git。更新正式网站时运行：

```powershell
npm --prefix web run deploy
```
